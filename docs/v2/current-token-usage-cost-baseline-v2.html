1Executive Summary
Model: claude-sonnet-4-6
This report captures the complete observed AI cost baseline for SnapNotes. All thirteen AI actions have now been measured using production-equivalent token counts and Sonnet 4.6 pricing.

Headline finding: Every measured action came in significantly below the unit economics reference cost — ranging from −59.5% to −93.4% cheaper than modelled. The actual gross margins across all measured actions are healthy, confirming the original pricing model was consistently conservative.

Caution: These are single-sample observations from controlled test scenarios, not averaged production logs. Variance across real user sessions may differ from these baseline measurements.

4Analysis: What the Data Shows
1Reference model was consistently conservative across all 13 actions
Every single measured action came in below its Sonnet 4.6 reference cost — none exceeded it. The original unit economics model was built on worst-case token assumptions (e.g. 16K output for Prepare Guide, 3 full tutor turns for Practice). Real usage produces shorter, more focused outputs. The delta range is −51% (Exam Generation) to −98.9% (Diagram). This is universally good news for margins but confirms the reference model needs updating before it can be used for future planning.
2Gross margin reality is far better than the reference implied
The unit economics model showed Course Generation at −153% gross margin (charging 1 LU for a $0.63 action). With actual LU pricing (10 LU = $2.50 revenue) and observed cost of $0.197, the real margin is 92.1%. Similarly, HW Smart was modelled at −39.6% but is actually 97.3%. The LU pricing table is extremely well-calibrated for revenue generation, even if the cost reference was overstated. Every action with a defined LU price produces a margin above 82%.
3Diagram is the cheapest AI action — but has a mandatory floor cost
Diagram Claude tokens cost only $0.0018 (−98.9% vs reference), but the flat $0.04 Recraft image generation fee is unavoidable. Total real cost is ~$0.0418. At 3 LU ($0.75 revenue), gross margin is still 94.4%, but unlike all other actions, Diagram has a cost floor that cannot be reduced through prompt optimisation alone. This makes it the only action where token efficiency improvements have limited impact on total cost.
4Course Generation: text vs upload input diverge significantly
Text-content generation produced 11,999 output tokens ($0.1973), while upload-file generation produced only 5,250 output tokens ($0.0958) — a 51% cost difference for the same 10 LU charge. This is the single largest cost lever in the measured data. If text-based inputs could be pre-processed to match the structure of file-based inputs, the average Course Generation cost could be materially reduced. At current LU pricing, both variants yield excellent margins (92.1% and 96.2%), but the gap represents an optimisation opportunity.
5Flashcard Batch is the lowest-cost action — but only 1 lesson was measured
The $0.0026 observed cost covers only 1 lesson (4 parallel calls aggregated). A full course has multiple lessons, so the actual per-action cost would be higher and proportional to lesson count. The reference cost ($0.027) likely assumed a full course batch. For a 5-lesson course, total expected cost would be ~$0.013 — still significantly below the $0.027 reference but not as extreme as the single-lesson measurement suggests. The per-lesson cost is approximately $0.0026.
6Chat (Prepare Guide) has cache write active — Chat (Course) does not
Chat (Prepare Guide) logs 3,317 cache write tokens at $3.75/M, adding a small premium on the first call. However, subsequent calls hitting that cache would read at $0.30/M — a 90% reduction on the cached portion. Chat (Course) has zero cache activity; for multi-turn course chat sessions where the system prompt and course content are stable, adding prompt caching could reduce per-turn input cost from $3.00/M to $0.30/M on the cached portion. Estimated saving: 40–60% per follow-up turn in multi-turn sessions.
7Lesson Expansion and Walkthrough still have no LU pricing entry
Both actions have confirmed observed AI costs (Lesson Expansion $0.0139, Walkthrough $0.0230) but no corresponding LU charge defined in the pricing table. This means they are currently either free to the user or bundled invisibly into another LU charge. This is a revenue gap — at $0.0139/call for Lesson Expansion, pricing it at 0.5 LU ($0.125 revenue) would yield a gross margin of ~89%. Walkthrough at $0.0230 could be priced at 1.0 LU ($0.25) for an 90.8% margin. Both must be assigned LU values before the billing system goes live.
8Homework Check (Standard) vs Smart: similar observed cost, different pipeline
Standard Homework Check observed $0.0284 vs Smart at $0.0270 — nearly identical actual cost despite the Smart mode using a more sophisticated decompose + solve pipeline. The Smart mode produced significantly more output tokens (1,686 vs 1,602) but with far fewer input tokens (575 vs 1,460), suggesting the Smart pipeline is more focused in its prompting. Both are dramatically cheaper than the reference ($0.237 standard, $0.349 smart). Standard: −88%, Smart: −92.3%.
5What Can Be Improved
HIGH
Assign LU pricing for Lesson Expansion and Walkthrough
Both actions have confirmed AI costs ($0.0139 for Lesson Expansion, $0.0230 for Walkthrough) but no LU entry in the pricing table. They are either being given away for free or silently bundled. Recommended pricing: Lesson Expansion at 0.5 LU ($0.125 revenue, ~89% margin); Walkthrough at 1.0 LU ($0.25 revenue, ~90.8% margin). This must be resolved before the billing system goes live.
Impact: Revenue uplift on currently untracked AI spend. Fixes a gap in the unit economics model.
HIGH
Replace single-sample observations with averaged production data
All current observed costs are single-sample measurements from controlled test scenarios. A single course generation run is not representative — content length, complexity, and user behaviour vary significantly. Before treating these figures as the optimisation baseline, collect at least 20–50 representative requests per action from production and use the p50 (median) as the baseline cost. The Flashcard Batch result is particularly unreliable since only 1 lesson was measured.
Risk reduction: Prevents optimising against a non-representative baseline. Required for credible post-optimisation comparison.
HIGH
Prompt compression for Course Generation (Text) — output token reduction
Course Generation (Text) produced 11,999 output tokens — the highest of any measured action and the primary cost driver at $0.1973. The upload-based variant produced only 5,250 output tokens ($0.0958) for the same 10 LU charge — a 51% cost difference. Applying structured output constraints, JSON schema enforcement, or output length caps to the text-based pipeline could reduce output tokens by 30–50%, bringing cost closer to the upload variant.
Estimated saving: ~$0.05–$0.10 per text-based course generation (25–50% of $0.1973).
MEDIUM
Extend prompt caching to Course Chat and Walkthrough
Chat (Prepare Guide) already writes 3,317 cache tokens, but Chat (Course) has zero cache activity. For multi-turn course chat sessions where the system prompt and course content are stable across turns, adding a cache write on the first turn and cache read on subsequent turns would reduce per-turn input cost from $3.00/M to $0.30/M for the cached portion. Similarly, Walkthrough uses ephemeral cache on the system prompt — extending this to include the question context could reduce costs on repeat walkthroughs for the same problem.
Estimated saving on Chat (Course): ~40–60% per follow-up turn in multi-turn sessions.
MEDIUM
Flashcard Batch: measure full-course generation before setting baseline
The current flashcard observation ($0.0026) covers only 1 lesson. A full 5-lesson course would generate ~$0.013 in flashcard AI costs — still well below the $0.027 reference but not as low as the single-lesson figure implies. Before using the flashcard cost in any optimisation analysis, trigger a full course flashcard generation and sum all parallel call costs. The per-lesson cost of ~$0.0026 is a useful unit, but the per-action cost must reflect a full generation run.
Accuracy: Prevents underestimating the true per-action flashcard cost by 5×.
MEDIUM
Update the unit economics reference model to reflect actual observed costs
The current reference model overestimates costs by 51–99% across all 13 measured actions. Once the baseline is finalised (after collecting production-averaged data), the unit economics reference table should be updated to use observed costs as the new planning baseline. This ensures future pricing decisions and LU allocation decisions are built on reality rather than conservative assumptions from the original model.
Impact: More accurate LU pricing decisions and margin forecasting going forward.
LOW
Investigate Diagram token footprint — only 8 output tokens observed
The Diagram action produced only 8 output tokens in this test — extremely low and likely reflects a simple formula diagram from the algebra practice session. More complex diagrams (geometry, physics, chemistry) may produce significantly more tokens. The $0.04 Recraft fee is the dominant cost regardless, but it would be worth measuring 3–5 different diagram types to understand whether the token cost is consistently negligible or varies with complexity.
Risk: Current $0.0018 token cost may be unrepresentative for complex diagrams.
